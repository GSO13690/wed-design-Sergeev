When HTML5 has become a standard?
HTML5 was first released in a public-facing form on 22 January 2008, with a major update and “W3C Recommendation” status in October 2014. HTML5 is the latest version of the HTML standard, which defines the structure and semantics of web pages. HTML5 introduces new features and elements, such as multimedia support, semantic markup, offline storage, and APIs. HTML5 is designed to be backward compatible with previous versions of HTML, and to work across different browsers and devices.

What is a difference between W3C HTML5 and WHATWG HTML Living Standard?
The difference between W3C HTML5 and WHATWG HTML Living Standard is that the former is a fixed version of the HTML specification, while the latter is a continuously evolving document that reflects the current state of the web. The W3C HTML5 specification was finalized in 2014 and is now being followed by HTML5.1

What is the minimal HTML page code?
The minimal HTML page code is the smallest possible HTML document that still validates as HTML5.

Comment on semantic and presentational markup. Why presentational markup is considered deprecated now?
Semantic markup is the subset of HTML that describes the content and structure of a document, whereas presentational markup is the subset of HTML that is used to determine the appearance of the document. Semantic markup is important for accessibility, search engine optimization, and making use of the built-in features browsers provide for content to work optimally2. Presentational markup, on the other hand, is considered deprecated because it mixes content with presentation, making the code less maintainable, readable, and semantic3. Presentational markup also violates the principle of separation of concerns, which states that HTML should only be used for structure, CSS for presentation, and JavaScript for behavior. Therefore, presentational markup should be avoided and replaced with semantic markup and CSS whenever possible.

How you can check browser support for particular tag or attribute?
There are different ways to check browser support for a particular HTML tag or attribute. One way is to use a reference website like W3Schools or Can I use, which provide tables of browser compatibility for various HTML features. Another way is to use a JavaScript library like Modernizr, which can detect the availability of specific HTML features in the user’s browser. A third way is to use a conditional comment, which is a special syntax that only certain browsers can interpret.

What are the audio and video formats, officially supported by HTML standard? What happens if you use other format?
The HTML standard officially supports the following audio and video formats: Audio: MP3, WAV, and OGG Video: MP4, WebM, and OGG These formats are supported by the "audio" and "video" elements, which allow you to embed audio and video files on a web page without requiring plug-ins12. If you use other formats, such as WMA, AAC, or FLV, the browser may not be able to play them, or it may require a plug-in or an external player. Different browsers may also have different levels of support for different formats, so it is recommended to provide multiple sources with different formats for the same media file.
